# 🚲 런던 자전거 수요 예측 프로젝트

## 📌 개요

- **목표**: 날씨, 시간대, 계절 등 변수를 기반으로 런던 자전거 공유 수요 예측
- **데이터**: London Bicycle Sharing 데이터 (17,414 rows, 2015~2016년 시간별)
- **문제 유형**: 시계열 회귀 (ML + DL 비교)
- **사용 도구**: Python (scikit-learn, XGBoost, LightGBM, Keras, statsmodels)

---

## 📊 데이터 구성

| 변수 | 설명 |
|------|------|
| cnt | 자전거 대여 수 (타겟) |
| t1 / t2 | 실제 온도 / 체감 온도 (°C) |
| hum | 습도 (%) |
| wind_speed | 풍속 (km/h) |
| weather_code | 날씨 코드 (맑음~눈) |
| is_holiday / is_weekend | 공휴일/주말 여부 |
| season | 계절 (0=봄, 1=여름, 2=가을, 3=겨울) |

---

## 📊 분석 프로세스

### 1. 전처리

- 시간 관련 피처 추가: `year`, `month`, `hour`, `weekday`
- 시간대별 3σ 기준 이상치 제거 (17,414 → 17,265 rows)
- 범주형 변수 더미화 → 최종 57개 피처

### 2. ML 모델 비교 (RMSE 기준)

| 모델 | RMSE |
|------|------|
| DNN (4층: 160→60→20→1) | 254.5 |
| **XGBoost** | **300.17** |
| Random Forest | 302.29 |
| LightGBM | 319.68 |

> DNN이 가장 낮은 RMSE를 기록

### 3. 시계열 모델 비교 (RNN / LSTM / GRU)

| 모델 | RMSE |
|------|------|
| **RNN** | **125.84** |
| LSTM | 289.94 |
| GRU | 351.98 |

- MinMaxScaler 스케일링 후 시계열 슬라이딩 윈도우로 학습
- RNN이 단순 구조임에도 시계열 예측에서 가장 우수한 성능

---

## 💬 인사이트

- 시간대(`hour`)별 수요 패턴이 뚜렷함 (출퇴근 시간 피크)
- PACF 기준 lag=1이 가장 강한 자기상관 → 단기 예측에 RNN 구조 적합
- ML 대비 시계열 전용 모델(RNN)이 RMSE에서 약 50% 이상 개선
